# 要件定義書

**プロジェクト名：** 個人タスク管理アプリ  
**作成日：** 2026-05-11

---

## 1. システム概要

### 1.1. システム構成

```
[Streamlit] → HTTP → [FastAPI] → [JSONファイル]
```

### 1.2. 用いる技術

| レイヤー | 技術 |
|---|---|
| フロントエンド | Streamlit |
| バックエンド | FastAPI | 
| データの保存 | JSONファイル |

---

## 2. 機能要件

### 2.1. タスク登録機能

- ユーザーがテキストエリアでタイトル、カテゴリ[仕事, プライベート, その他]、優先度[高, 中, 低]を入力できる。
- タイトルが空の場合、登録できない。
- 登録後は入力欄を空にする。

### 2.2. 一覧表示機能

- 登録済みタスクを日付の降順（新しい順）で一覧表示する
- タスクのタイトルを表示する。タイトル以外の情報は、一覧表示機能では表示しない。

### 2.3. 詳細表示機能

- 各タスクのID（登録順）、日付、タイトル、カテゴリ、優先度、ステータス[未完了, 完了]を表示する。

### 2.4. 完了更新機能
- タスクのステータスの初期状態は[未完了]だが、そこから[完了]に更新できる。
- 4.に示すタスク詳細画面より、この機能を実行できる。

---

## 3. 非機能要件

### 3.1. 性能
- JSONファイルに保存されているタスクが1,000件以下の場合において、支障なく動作すること。

### 3.2. 信頼性
- APIエラー時は、タスク情報送信におけるエラーが起きたという旨のメッセージを表示すること。

### 3.3. 保守性
- フロントとバックとJSONは別ファイルで管理すること。

## 5. API一覧

| メソッド | パス | 機能 |
|---|---|---|
| GET | /tasks | タスク一覧を返す |
| GET | /tasks/{id} | 指定IDのタスク詳細を返す |
| POST | /tasks | タスクをJSONファイルに保存する |
| PUT | /done | タスクのステータスを完了にする |

---

## 6. データ仕様

```json
[
    {
        "id": 0001,
        "created_at": "2026-05-11T10:00:00+09:00",
        "content": "XXさんに14:30に電話をかける",
        "category": "仕事",
        "priority": "高",
        "status": "未完了"
    }
]
```

| フィールド名 | 型 | 内容 |
|---|---|---|
| id | 整数 | タスクID（自動採番） |
| created_at | 文字列（ISO 8601） | 登録日時 |
| content | 文字列 | タスクのタイトル |
| category | 文字列 | カテゴリ |
| priority | 文字列 | 優先度（高／中／低） |
| status | 文字列 | ステータス（完了／未完了） |
