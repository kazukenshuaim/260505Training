## 3. 画面構成とレイアウト

### 3.1. 画面一覧

| 画面名 | 説明 |
|---|---|
| タスク入力画面 | タスクを入力し登録する |
| タスク一覧画面 | 登録済みのタスクを一覧表示する |
| タスク詳細画面 | 特定のタスクの詳細を表示する |

### 3.2. 画面レイアウト

#### 3.2.1. タスク入力画面

ユーザーがテキストエリアでタイトルを入力、カテゴリ[仕事, プライベート, その他]と優先度[高, 中, 低]を選択できる。タイトルが空の場合、登録できない。また、登録後は入力欄を空にする。

```
┌─────────────────────────────┐
│  個人タスク管理アプリ         │
│  ─────────────────────────  │
│  [サイドバー]                │
│   ・タスク入力               │
│   ・タスク一覧               │
│                             │
│  ## タスク入力               │
│  ┌───────────────────────┐  │
│  │ （テキストエリア）      │  │
│  └───────────────────────┘  │
|  カテゴリ選択 [仕事▽]        |
|  優先度選択 [高▽]            |
│  [登録する]                  │
└─────────────────────────────┘
```

#### 3.2.2. タスク一覧画面

登録済みタスクを日付の降順（新しい順）で一覧表示する。タスクのタイトルを表示する。タイトル以外の情報は表示しない。

```
┌─────────────────────────────┐
│  個人タスク管理アプリ         │
│  ─────────────────────────  │
│  [サイドバー]                │
│   ・タスク入力               │
│   ・タスク一覧               │
│                             │
│  ## タスク一覧               │
|    □XXさんにoo時に電話する    |　
|    □PCパスワードを更新する    |
|    □13日xx時にyyさんとアポ    |
|    .....                    |
└─────────────────────────────┘
```

#### 3.2.3. タスク詳細画面

各タスクのID（登録順）、日付、タイトル、カテゴリ、優先度、ステータス[未完了, 完了]を表示する。ページ下部のチェックボタンで、完了更新ができる。

```
┌─────────────────────────────┐
│  個人タスク管理アプリ         │
│  ─────────────────────────  │
│  [サイドバー]                │
│   ・タスク入力               │
│   ・タスク一覧               │
│                             │
│  ## タスク詳細               │
|    ID: 0001                 |
|    日付:2026-05-11-15:00     |
|    タイトル:XXさんにoo時に電話する|
|    カテゴリ:仕事             |
|    優先度:高                 |
|    ステータス:未完了          |
|    ✓ステータスを完了にする    |
└─────────────────────────────┘
```

## 4. 画面遷移

タスク入力画面からタスク詳細画面は直接遷移することはできない。

```
タスク入力画面
       |
       | サイドバーで「タスク一覧」を選択
       ↓
タスク一覧画面
       |
       | 登録済みタスクのタイトルを押す
       ↓
タスク詳細画面
       |
       | サイドバーで「タスク一覧」を選択
       ↓
タスク一覧画面


タスク一覧画面またはタスク詳細画面
       |
       | サイドバーで「タスク入力」を選択
       ↓
タスク入力画面
```

## 5. 処理フロー

## 5.1. タスク入力、登録のフロー

```
[ユーザー] タスクタイトルを入力、カテゴリと優先度を選択して「登録する」を押す
        ↓
[Streamlit] 入力が空でないかチェック
        |
        |（空の場合はエラー表示して停止）
        ↓
[Streamlit] POST /tasks にリクエスト送信
        |
        |（FastAPI接続エラーの場合、HTTPエラーコード500を出力して停止）
        ↓
[FastAPI] リクエストを受け取る
        ↓
[FastAPI] タスクを JSON ファイルに保存
        |
        |（dataフォルダまたはJSONファイルがなければ作る）
        ↓
[FastAPI] 保存フラグをStreamlitに送信
        ↓
[Streamlit] 登録完了と表示する
```

## 5.2. タスク一覧表示のフロー

```
[ユーザー] サイドバーから「タスク一覧」を押す
        ↓
[Streamlit] GET /tasks にリクエスト送信
        |
        |（FastAPI接続エラーの場合、HTTPエラーコード500を出力して停止）
        ↓
[FastAPI] リクエストを受け取る
        |
        |（JSONファイルが空なら、タスクがないと表示）
        ↓                           
[FastAPI] JSONファイルのデータを降順に並べ替える
        ↓
[FastAPI] 降順データをStreamlitに返す
        ↓
[Streamlit] タスク一覧を表示
```

## 5.3. タスク詳細表示のフロー

```
[ユーザー] タスク一覧画面にて所望のタスクのタイトルを押す
        ↓
[Streamlit] GET /tasks/{id} にリクエスト送信する
        |
        |（FastAPI接続エラーの場合、HTTPエラーコード500を出力して停止）
        ↓
[FastAPI] リクエストを受け取る
        ↓
[FastAPI] JSONファイル内の指定IDに該当するタスクのデータをStreamlitに返す
        ↓
[Streamlit] 指定IDに該当するタスクの詳細を表示
```

## 5.4. タスク完了更新のフロー

```
[ユーザー] タスク詳細表示画面にて
        ↓
[Streamlit] PUT /done にリクエスト送信
        |
        |（FastAPI接続エラーの場合、HTTPエラーコード500を出力して停止）
        ↓
[FastAPI] リクエストを受け取る
        ↓
[FastAPI] 指定IDのタスクのデータの"status"を"未完了"から"完了"に更新する
        ↓
[FastAPI] 更新後データをStreamlitに返す
        ↓
[FastAPI] タスク詳細画面のステータスに"完了"を表示
```

## 6. データ設計

### 6.1. JSONファイルの構造サンプル

保存するデータはJSONファイルであるが、その構造サンプルを示す。

```json
[
    {
        "id": 0001,
        "created_at": "2026-05-11T10:00:00+09:00",
        "content": "XXさんに14:30に電話をかける",
        "category": "仕事",
        "priority": "高",
        "status": "未完了"
    }
]
```
### 6.2. 各フィールドの仕様

各フィールドの仕様を示す。ただし、statusは、生成後にタスク詳細画面にて変更可能。

| フィールド名 | 型 | 内容 | 生成元 |
|---|---|---|---|
| id | 整数 | タスクID（自動採番） | 登録時にバックエンドで自動生成 |
| created_at | 文字列（ISO 8601） | 登録日時 | 登録時にバックエンドで自動生成 |
| content | 文字列 | タスクのタイトル | ユーザー入力 |
| category | 文字列 | カテゴリ | ユーザー入力 |
| priority | 文字列 | 優先度（高／中／低） | ユーザー入力 |
| status | 文字列 | ステータス（完了／未完了） | 登録時にバックエンドで"未完了"と生成される |

## 7. エラー処理方針

| エラーの種類 | 発生箇所 | 対処方法 |
|---|---|---|
| 入力が空 | Streamlit | エラーメッセージを表示し、登録しない |
| dataフォルダが存在しない | FastAPI | dataフォルダを新しく生成して処理を続行する |
| JSONファイルがdataフォルダに存在しない | FastAPI | 空リストとして新しくファイル生成して処理を続行する |
| FastAPI接続エラー | FastAPI | HTTPエラーコード500を出力 |
